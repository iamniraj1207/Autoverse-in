Architecture Blueprint v1.0

AUTOVERSE
SYSTEM DESIGN

Complete technical architecture for the Automotive & F1 Enthusiast Platform — Flask + SQLite + Hybrid API

01 — Technology Stack

Three-Layer Architecture

AutoVerse uses a classic but powerful three-tier architecture. Each layer has a single responsibility.

LAYER 01

Frontend

HTML · CSS · Vanilla JS

Jinja2 templates handle page renders. Vanilla JS handles dynamic interactions — comparison engine, garage add/remove, timeline rendering.

LAYER 02

Backend

Python · Flask · Jinja2

Flask serves both HTML pages (Jinja routes) and JSON data (API routes). Session management handles auth state across requests.

LAYER 03

Database

SQLite · cs50 library

Lightweight relational database. Six core tables. Use the CS50 library's db.execute() for all queries — clean and CS50-familiar.

05 — Data Flow

Request Lifecycle

Two flows exist — one for page loads, one for dynamic actions.

→

FLOW B — DYNAMIC ACTION (API + JS)

User Action (JS)
→
fetch('/api/...')
→
Flask API route
→

→
JS renders UI
Comparison engine example: User checks 3 cars → JS collects IDs → fetch('/api/compare', {method:'POST', body: JSON.stringify({ids:[1,3,7]})}) → Flask queries DB → returns JSON → JS builds side-by-side table dynamically. No page reload.

07 — Critical Implementation Notes

Things That Will Save You

✓ Seed data first

Write seed.py before building any templates. You need real data to see if things work. Aim for 25 cars, 12 drivers, 6 teams, 30+ timeline events.

✓ Use the CS50 library

from cs50 import SQL — use db.execute() for all queries. It returns lists of dicts, which Jinja and JSON both handle perfectly.

⚠ Always check session

Every API endpoint that touches the garage must check session.get("user_id"). Return 401 if not logged in. Use the @login_required decorator.

⚠ UNIQUE constraint on garage

Add UNIQUE(user_id, car_id) to the garage table. Use INSERT OR IGNORE so double-adding doesn't crash the app.

→ Comparison UX pattern

On the cars page, add a checkbox to each card. A sticky bottom bar shows "Compare (2 selected)" and a button triggers the comparison. Don't use a form — use JS to collect IDs.

→ Timeline JS pattern

On driver/brand pages, call fetch('/api/timeline/driver/1') on page load. Render the returned events as a vertical timeline in the DOM. Sorted by year ascending.
